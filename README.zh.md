 TicketGuard-Fivetran Track

## **一、解决的真实问题**

FIFA 2026 世界杯门票单价极高（决赛票面价超 $2,000，黄牛价可达 $8,000+），二手市场欺诈极其普遍。**普通买家在点击"Buy"之前，没有任何工具帮他判断这个页面/这张票是否可信。**

买家面临三类核心风险：

| 欺诈类型 | 手法 | 买家痛点 |
| --- | --- | --- |
| **仿冒售票网站** | 克隆 StubHub/Ticketmaster 页面，域名微调（如 stubhub-tickets.com），诱导用户输入支付信息 | 页面视觉几乎一致，普通用户无法识别 |
| **视线受阻座位欺诈** | 隐瞒"Obstructed View"座位（前方有承重柱、转播摄像机），当正常票出售 | 官方售票页警示极隐蔽，买家买前无法看到真实视线 |
| **非法溢价欺诈** | 利用信息差在私下渠道完成超限高价交易（纽约州限制溢价不超面值 10%） | 买家不了解各州《反黄牛法》，无法判断当前价格是否违法 |

**TicketGuard** 是一个**购票前实时介入**的多步骤 Agent。用户在浏览器打开任意卖票页面（第三方平台挂牌、私下卖家链接、或疑似仿冒网站），只需把 URL 粘贴给 Agent，在点击"Buy"之前完成：网站可信度审计 → 座位视线评估 → 实时市场比价 → 法规合规检查 → 综合购买建议。

> **核心定位**：不是事后验票，而是**购票决策前的实时守门员**。参考 PhishDetect 的沙盒浏览器主动访问模式，Agent 主动抓取并分析卖票页面，用户无需上传任何文件。

## **五、世界杯专属优化（差异化特色）**

通用钓鱼检测工具（PhishDetect、SentinelEye 等）只做网站真伪判断，TicketGuard 在此基础上叠加了三层世界杯专属能力：

### **特色 1：FIFA 2026 场馆视线数据库**

- 覆盖全部 16 个 FIFA 2026 场馆（美国 11 个 + 加拿大 2 个 + 墨西哥 3 个）
- 专门标注世界杯转播摄像机机位（每场比赛摄像机位置固定，是世界杯特有遮挡源）
- 通过 Fivetran 同步 View From My Seat 众包实拍图，提供"坐在该位置的真实视角"

### **特色 2：三国法规合规引擎**

- 美国各州《反黄牛法》溢价上限各不相同（纽约 10%、科罗拉多无上限等）
- 加拿大各省票务法规
- 墨西哥联邦消费者保护法
- Agent 根据场馆所在地自动匹配对应法规，给出"这个价格在当地是否违法"的明确判断

### **特色 3：世界杯赛程联动比价**

- 比价不只看"同区域均价"，还考虑赛事重要性权重
- 小组赛 vs 淘汰赛 vs 决赛的溢价基准完全不同
- Fivetran 同步 wc26-mcp 赛程数据，Agent 自动识别当前场次的赛事阶段，给出合理溢价区间参考

## **六、Fivetran MCP 数据管道设计（Partner Power 核心）**

Fivetran 是本方案的"数据超能力"来源。Agent 通过 Fivetran MCP Server 按需触发三条数据管道，将异构外部数据实时同步至 BigQuery，无需手写 ETL 脚本。

### **管道 1：场馆座位坐标与视线数据**

| 字段 | 说明 |
| --- | --- |
| `venue_id` | 场馆唯一标识（16 个 FIFA 2026 场馆） |
| `section` / `row` / `seat` | 座位三级坐标 |
| `obstruction_type` | 遮挡类型（柱子 / 摄像机 / 玻璃边框 / 无） |
| `obstruction_pct` | 视线遮挡百分比 |
| `crowdsource_img_url` | View From My Seat 众包实拍图 URL |
| `goal_distance_m` | 座位到最近球门直线距离（米） |

**数据源**：View From My Seat（众包平台）+ FIFA 官方场馆 CAD 数据 **触发时机**：Step 3 解析出座位号后，按需触发增量同步

---

### **管道 2：官方票价、坐席分级与法规底表**

| 字段 | 说明 |
| --- | --- |
| `match_id` | 赛事 ID |
| `venue_id` | 场馆 ID |
| `section` | 看台区域 |
| `official_category` | 官方等级（Cat 1 / Cat 2 / Cat 3 / VIP） |
| `face_value_usd` | 官方面值（美元） |
| `match_stage` | 赛事阶段（小组赛 / 淘汰赛 / 决赛） |
| `resale_legal_cap_pct` | 所在州/省法定溢价上限（%） |

**数据源**：FIFA 官方售票 API + 各州/省法规数据库 + wc26-mcp 赛程数据 **触发时机**：Step 5 合规检查时调用

---

### **管道 3：实时二手市场挂牌价监控**

| 字段 | 说明 |
| --- | --- |
| `match_id` | 赛事 ID |
| `section` | 看台区域 |
| `listing_price_usd` | 挂牌价（美元） |
| `seller_id_hash` | 卖家匿名 ID（检测同一卖家多票） |
| `listing_count_same_seat` | 同座位号当前挂牌数量（>1 则预警） |
| `p25` / `p50` / `p75` | 同区域价格分位数 |
| `updated_at` | 数据更新时间戳 |

**数据源**：StubHub API + Viagogo API **触发时机**：Step 4 比价时调用

## **九、参考项目与论文**

### **购前实时网站识别（核心参考）**

1. **PhishDetect**（Devpost）
    - 链接：devpost.com/software/phishdetect-79j85p
    - 借鉴方法：**沙盒浏览器主动访问**。Agent 控制沙盒浏览器像真实用户一样访问可疑链接，实时拦截网络请求，识别仿冒行为。这是"主动访问 + 实时判断"而非"被动接收截图"，是我们 Step 1 网站可信度审计的直接技术原型。
2. **SentinelEye**（Devpost）
    - 链接：devpost.com/software/sentinel-eye
    - 借鉴方法：**多模态视觉安全巡检**。Agent 像真实用户一样导航网站，用多模态 AI 识别安全漏洞。借鉴其"Agent 主动操控浏览器 + Gemini Vision 截图分析"的组合方式。
3. **CLASP**（arxiv 论文）
    - 链接：arxiv.org/html/2510.18585
    - 借鉴方法：**多 Agent 分工分析**。多个 LLM Agent 分别负责 URL 结构分析、页面内容分析、视觉相似度分析，最后汇总判断。我们的 6 步工作流借鉴此分工思路，每步 Agent 专注单一维度。

### **欺诈检测通用方法**

1. **PhishAgent**（AAAI 2025 Oral）
    - 论文：arxiv.org/abs/2408.10738
    - 借鉴方法：**双源比对架构**。品牌视觉规范知识库 vs 截图对比，我们将其迁移为"官方场馆/票价数据库 vs 页面抓取数据"的比对逻辑。AAAI 2025 Oral 级别，技术可信度高。
2. **MultiPhishGuard**（arxiv 2025）
    - 论文：arxiv.org/abs/2505.23803
    - 借鉴方法：**多 Agent PPO 加权聚合**。多个 LLM Agent 输出用强化学习权重聚合，达到 97.89% 准确率。我们的 5 维度评分聚合参考此思路，不同维度权重可根据场景调整（网站可信度权重最高）。
3. **Large Multimodal Agents for Phishing Detection**（arxiv）
    - 论文：arxiv.org/html/2412.02301v1
    - 核心结论：Gemini 1.5 Flash 同时分析 URL + 网页截图，无需训练专用模型即可达到高精度，且成本比纯多模态方式低 2.6 倍。直接支撑我们用 Gemini 3 做页面分析的技术选型和成本可行性。
4. **FinGuard**（Devpost）
    - 链接：devpost.com/software/finguard-hr0apk
    - 借鉴方法：**图片化攻击识别 + 并排对比展示**。借鉴其"原始页面 vs AI 标注版并排展示"的 Demo 视觉设计，以及对嵌入图片中的伪造内容的像素级检测思路。
5. **SafetyKit**（商业产品）
    - 链接：safetykit.com
    - 借鉴方法：**卖家网络关联检测**。通过 Fivetran 管道 3 检测"同一座位号被多个卖家同时挂牌"这种批量造假信号，参考 SafetyKit 的跨实体关联分析思路。

### **世界杯数据底座**

1. **wc26-mcp**（GitHub）
    - 链接：github.com/jordanlyall/wc26-mcp
    - 借鉴方法：直接接入其场馆数据结构、赛程 API 字段和城市指南工具，作为赛事数据底座。
2. **jennyslu/fraud-detection-app**（GitHub）
    - 链接：github.com/jennyslu/fraud-detection-app
    - 借鉴方法：票务欺诈场景的特征工程——价格异常、卖家行为模式、座位号重复出现频率等强信号字段设计。
